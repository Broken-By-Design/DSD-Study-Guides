## Contributing!
### branch layout
We prefer to work on branches with tags like:
- Feat/
- Fix/
- etc...

We Always push back to the dev branch before pushing to main  
This ensures no broken code ends up on main as we test before every Pr to main!  

### Commit messages
We like to use tags ate the start of commit messages like:
- UPD:
- ADD:
- REM:
- FIX:
- etc...

This keeps the logs clean and easy to see what changed at a glance  
No need to do this, Tho it would be preferable to us!
